Objectifs du projet

- avoir un container sur lequel la BDD est disponible (on peut faire des requêtes pour acceder à des données, des requêtes pour modifier la donnée, des requêtes pour ajouter de la donnée)

- avoir un container pour faire le preprocessing et le cleanning des données (ce container produit un csv à partir de de la BDD. le csv produit présente une game par ligne. le vecteur est utilisable par des algos de ML)

- avoir un container qui fait tourner notre model (un container par par model)

- avoir un container qui permet d'entrainer notre model (et met à jour le model si le nouveau est meilleur)

- un container pour ML flow / air flow

- un container qui permet de collecter de nouvelles games (sous quelles conditions ? les games d'un tel invocateur ?)
